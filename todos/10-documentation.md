# 10 - Documentation

## Goal

Write clear documentation for users and contributors.

## Tasks

- [ ] Add docstrings to all public functions and classes
- [ ] Write a quickstart guide in README (install, first evaluation, comparison)
- [ ] Document each metric: definition, formula, interpretation, source paper
- [ ] Document supported tokenizer formats (HF, tiktoken, sentencepiece)
- [ ] Document how to add custom corpora / datasets
- [ ] Add usage examples in an `examples/` folder
- [ ] Add CONTRIBUTING.md with dev setup and code style
- [ ] Add type hints to all public API

## Acceptance Criteria

- New user can go from `pip install toklens` to first result in under 5 minutes
- Every metric has a clear explanation of what it measures and why it matters
- Contributing guide covers dev setup, testing, and PR process
